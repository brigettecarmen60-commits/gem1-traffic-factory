# 案例展示 · Gem1 流量工厂

> 三个真实赛道，从一句话到 10 条选题的完整过程。
> 所有案例均使用 Mode B（默认），快速模式输出。

---

## 案例 1：全屋定制

**输入**："我是做柜子的。"

**模式**：Mode B 结构化荒诞版

**为什么选 Mode B**：用户给的信息极少（只有 5 个字），Mode A 需要一手行业底料才能跑。Mode B 专为这种情况设计——一句话就够了。

**亮点**：系统自动把"做柜子"拆成了物理切片（柜门/板材/封边/安装/量尺），然后用三维错位老虎机逐一制造错位。选题不是讲"我的柜子多好"，而是从外行人会关心的角度切进去。

→ [完整 10 条选题输出](output-examples/quwudingzhi.md)

## 案例 2：减脂餐

**输入**："做减脂餐的，想吸引更多想健康饮食的人。"

**模式**：Mode B 结构化荒诞版

**为什么选 Mode B**：减脂餐赛道极其拥挤，常规选题（"今天吃什么""低卡食谱"）已经没有差异化空间。需要荒诞错位来破圈。

**亮点**：系统把"减脂餐"拆成物理切片（鸡胸肉/沙拉/餐盒/称/冰箱），然后匹配了 Code 12（替你测试）和 Code 58（杀鸡牛刀）制造视觉奇观。

→ [完整 10 条选题输出](output-examples/jianzhican.md)

## 案例 3：宠物用品

**输入**："卖宠物用品的，淘宝店。"

**模式**：Mode B 结构化荒诞版

**为什么选 Mode B**：宠物用品的内容赛道被"萌宠"统治——只有猫狗卖萌，没有产品出现。Mode B 可以制造"离谱评测"和"荒诞测试"让产品进入画面而非广告。

**亮点**：系统大量使用 Code 11（叫外援——让宠物自己当评测员）和 Code 12（替你测试——用极端实验测产品），让产品在宠物的"自然行为"中被展示。

→ [完整 10 条选题输出](output-examples/chongwuyongpin.md)

## 使用建议

1. **先看快速模式**：快速模式 5 字段够你判断"这条选题值不值得拍"
2. **需要深入理解时再看深度模式**：15 字段告诉你"为什么这条选题能成立"
3. **一个赛道至少跑 2-3 次**：老虎机每次的 A×B×C 组合不同，多跑几次选最好的
4. **不要只选"最顺眼"的**：选那个"我看了有点紧张但又想试试"的——那才是破圈的信号
